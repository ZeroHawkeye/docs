### 介绍

了解我们的客户——不仅是他们是谁，还有他们做什么——对于有效地定制我们的产品和服务至关重要。在运行自助服务模式时，会有很多客户进门，而我们对他们几乎一无所知。主动了解这些人是谁的过程传统上是耗时的，涉及手动数据收集和分析以获取可行的洞察。

然而，借助LLM及其高级数据提取能力，我们已经自动化了这个过程。使用LLM提取和分析客户数据，我们显著减少了工作量，使我们能够比以往更有效地理解和服务我们的客户群。

如果您的技术知识有限，您可以构建一个自动化流程，获取有关客户的有针对性的信息，用于产品方向和潜在客户生成。以下是如何使用[Make](https://make.com/)和[Firecrawl](https://www.firecrawl.dev/)自己完成这项工作。

---

### 工具概述

**Firecrawl**

Firecrawl是一个用于抓取、搜索和提取的平台。它允许您从网络获取数据并将其转换为LLM可读的markdown或结构化数据。

当我们想要获取有关客户的信息时，我们可以使用Firecrawl的LLM提取功能来指定我们想从他们的网站获取的特定信息。

**Make.com（前身为Integromat）**

Make是一个自动化平台，允许用户创建自定义工作流程，无需深厚的技术知识即可连接各种应用程序和服务。它使用可视界面，用户可以拖放元素来设计他们的自动化流程。

我们可以使用Make将用户数据的电子表格连接到Firecrawl，只需一点JSON就可以进行提取。

太好了！现在我们已经配置好了，我们要准备设置HTTP请求。为此，我们将前往https://firecrawl.dev注册并获取API密钥（您可以免费开始！）。注册后，您可以前往https://firecrawl.dev/account查看您的API密钥。

我们将使用Firecrawl的Scrape端点。这个端点将允许我们从单个URL拉取信息，将其转换为干净的markdown，并用它来提取我们需要的数据。我将使用他们文档中的API参考填写我们Make HTTP请求中的所有必要条件。

现在在Make中，我使用Firecrawl的文档配置API调用。我们将使用POST作为HTTP方法，并有两个头部。

```
Header 1:
Name: Authorization
Value: Bearer your-API-key

Header 2:
Name: Content-Type
Value: application/json
```

![](https://i.imgur.com/LJ8g142.png)
我们还想设置我们的正文和内容类型。在这里我们将做：

```
Body type: Raw
Content type: Json (application/json)
```

我们还将点击"yes"来解析我们的响应。这将自动将我们的响应解析为JSON。

请求内容是我们想要实现的主要内容。以下是我们将用于此用例的请求内容：

```
{
  "url": "1. url(B)",

"pageOptions": {
    "onlyMainContent": true
  },
  "extractorOptions": {
    "mode": "llm-extraction",
    "extractionPrompt": "提取公司描述（用一句话解释公司做什么），公司行业（软件、服务、AI等）- 这实际上应该只是一个带有几个关键词的标签，以及他们服务的对象（谁是他们的客户）。如果没有明确的信息来回答问题，请写'无信息'。",
    "extractionSchema": {
      "type": "object",
      "properties": {
        "company_description": {
          "type": "string"
        },
        "company_industry": {
          "type": "string"
        },
        "who_they_serve": {
          "type": "string"
        }
      },
      "required": [
        "company_description",
        "company_industry",
        "who_they_serve"
      ]
    }
  }
}
```

![](https://i.imgur.com/DrMc1g2.png)

\*_注意_ 截图中的绿色字段是您可以在Make UI中选择的动态项。代替`url (B)`，该块可能是您数据中的第一个URL。

![](https://i.imgur.com/D4HCBNe.png)

太棒了！现在我们已经配置了HTTP请求。让我们测试一下，确保一切正常工作。在Make中点击"run once"，我们应该会收到数据返回。

![](https://i.imgur.com/QuQZs0U.png)

当我们运行时，让我们检查我们的第一个操作。在输出中，我们应该得到一个`status code: 200`，这意味着我们的API请求成功了。在输出中，点击数据以确保我们获得了所需的数据。

![](https://i.imgur.com/pm614VA.png)

我们的输出看起来成功了！在llm_extraction中，我们看到了我们想从网站获取的三个数据属性。

\*_注意_ 如果您在第一个操作上得到`500`错误，而在后续操作上得到`200`响应，这可能是因为操作试图在您数据的第一行（标题行）上执行。这将导致将数据导回表格时出现问题！确保如前所述从第二行开始。

现在我们知道HTTP请求正常工作，剩下的就是将Firecrawl输出的JSON放回我们的电子表格。

### 使用案例

LLM提取允许我们快速从网络获取与我们业务相关的特定信息。我们可以使用这些自动化来完成各种任务。

**产品：**
特别是对于自助服务公司，我们可以了解使用我们产品的行业趋势。使用我们技术的前2-3个行业是什么，他们用它做什么？这将使我们能够通过优先考虑正确的客户来做出更好的产品决策。

**业务发展：**
通过了解我们的用户是谁，我们可以寻找可能从我们的产品中受益的类似公司。通过进行类似的自动化，我们可以从可能从我们产品中受益的潜在客户中提取积极指标。

我们还可以使用这些数据生成更好的外联电子邮件，更具体地针对个别潜在客户。

**市场研究：**
市场研究公司花费大量时间进行二次研究，特别是在利基行业。我们可以通过自动化从多样化来源提取和组织数据来简化数据收集。这种自动化有助于提高效率并随着数据需求的增长而扩展，使其成为快速发展行业中战略决策的有价值工具。
