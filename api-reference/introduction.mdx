## 基础 URL

所有请求都包含以下基础 URL：

```bash
https://api.firecrawl.dev 
```


## 认证

进行认证时，需要包含一个 Authorization 头部。该头部应包含 `Bearer fc-123456789`，其中 `fc-123456789` 代表您的 API 密钥。

```bash
Authorization: Bearer fc-123456789
```
​

## 响应代码
Firecrawl 使用常规的 HTTP 状态码来表示请求的结果。

通常，2xx HTTP 状态码表示成功，4xx 代码表示与用户相关的失败，而 5xx 代码表示基础设施问题。

状态 | 描述
--- | ---
200 | 请求成功。
400 | 请验证参数的正确性。
401 | 未提供 API 密钥。
402 | 需要付款
404 | 找不到请求的资源。
429 | 已超过速率限制。
5xx | 表示 Firecrawl 服务器错误。

有关所有潜在 API 错误的详细解释，请参阅错误代码部分。

​
## 速率限制
Firecrawl API 有速率限制，以确保服务的稳定性和可靠性。速率限制适用于所有端点，并基于在特定时间段内发出的请求数量。

当您超过速率限制时，您将收到 429 响应代码。
